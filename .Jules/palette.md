## 2024-05-14 - Prevent Mobile Keyboard on Custom Keypads
**Learning:** When building an interface with a custom on-screen keypad (like a calculator) that uses a standard HTML `<input>` to display the result, mobile browsers will automatically focus the input and pop up the native mobile keyboard when the user taps on the display area. This severely disrupts the user experience, hiding the custom keypad they are supposed to use. Additionally, an input without context is unhelpful to screen reader users.
**Action:** Always add the `readonly` attribute to input elements serving solely as displays in custom keypad interfaces. This prevents the native keyboard from appearing on mobile devices while still allowing JavaScript to update the input's `.value`. Additionally, ensure it has a descriptive `aria-label` (e.g., `aria-label="Calculator display"`) to provide context to screen readers.
